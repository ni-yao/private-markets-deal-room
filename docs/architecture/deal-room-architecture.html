Key data & control flows

1Deal team → Container App SPA/API over HTTPS (Entra where required).
2Teams → Foundry agents → /mcp-persona on the Container App; each agent's key binds it to one persona server-side.
3Container App ↔ Cosmos DB for all deal reads/writes (managed-identity RBAC, _etag concurrency).
4Container App → Azure AI Foundry for chat/reasoning + the in-app persona tool loop.
5Container App → Azure AI Search (dealroomaisearch) for search_documents & get_crm grounding.
6Container App → Microsoft Fabric / OneLake for market intelligence & to archive SEC filings.
7Container App → SEC EDGAR to fetch filings, then persist to Storage & OneLake.
8Container App → Service Bus / Event Grid → Function App for deal-lifecycle events.
9Foundry sourcing agent → Grounding with Bing for live news signals.
10AI Search index ← Azure OpenAI integrated vectorizer (text-embedding-3-small).
11Container App → Key Vault (secrets) & → Microsoft Graph for Teams/SharePoint provisioning.
12All services → Application Insights / Log Analytics → Azure Monitor.
